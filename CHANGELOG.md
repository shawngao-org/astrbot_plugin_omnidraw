# Changelog

## 3.3.10

- 新增指令失败回复配置，支持自定义 `/画`、`/自拍` 和宏指令的报错提示。
- 失败文案与等待文案统一纳入 `reply_config`，原生配置页和 Pages WebUI 均可编辑。

## 3.3.9

- 新增图片缓存清理：WebUI 可一键清理 `temp_images` 与 `user_refs` 中的图片缓存，且不会触碰人设参考图。
- 新增定时清理与容量上限自动清理配置，原生配置页和 Pages 配置页均可设置。
- 新增管理员指令 `/清理缓存`，用于聊天侧手动释放临时图片缓存。

## 3.3.8

- 生成调度新增图像生成、人设自拍、视频渲染备用节点开关，Pages 配置页可按点击顺序多选备用节点。
- 图像与自拍链路继续按配置顺序兜底，视频渲染现在也会在主节点失败后逐个切换备用视频节点。
- 原生配置页链路说明补充主节点/备用节点顺序规则，并对重复节点做去重处理。

## 3.3.7

- 新增指令回复文案配置，支持自定义 `/画`、`/自拍` 与宏指令开始生成前的等待回复。
- 原生配置页和 Pages 配置页共用 `reply_config`，并内置默认文案与常用占位符。

## 3.3.6

- 合并旧版权限管理与每日额度系统：`allowed_users` 现在作为用户白名单使用，命中后不限次数。
- 新增用户黑名单、群组白名单和统一的权限优先级：黑名单禁止使用，用户/群白名单豁免限额，其余用户默认可用并按每日额度统计。
- 新增 `/签到` 指令，受限用户可每日签到获得额外生图额度，支持固定值或随机区间配置。
- Pages 用量面板展示用户签到奖励、签到时间、群号与权限层级，原生配置页和 Pages 配置页均可配置新权限项。

## 3.3.5

- 新增按 AstrBot 用户 ID 统计的每日生图限额，可在原生配置页和 Pages 配置页同时配置。
- 新增当日生图统计 API 与 Pages 用量面板，展示当天总张数和每个用户的生图张数。
- `/画`、`/自拍`、宏指令以及 LLM 生图工具会在成功产图后计入当日统计，并在超出限额时阻止继续生图。

## 3.3.4

- 修复 AstrBot 原生配置页在人设列表内部点击“管理文件”时提示 `Config item not found or not file type` 的问题。
- 避开 AstrBot 当前不支持的 `template_list` 内嵌 `file` 结构，改为“当前激活人设参考图上传入口”负责原生上传，并自动同步到当前 active persona。
- 增强 `files/...` 原生上传路径解析，确保原生配置页上传的参考图能被生图链路作为本地图片正常读取。

## 3.3.3

- 恢复 AstrBot 原生插件配置入口，`_conf_schema.json` 现在覆盖 Pages 配置页可编辑的权限、链路、Provider、人设、副脑、预设和调试配置。
- 新增原生配置与 Pages 配置的单源同步策略：Pages 保存会同步写回 AstrBot 原生配置；插件启动或检测到原生配置保存后会同步到运行态和本地持久化备份。

## 3.3.2

- 修复配置页临时预览 URL 被保存进多组人设配置，导致刷新后显示“1 图”但预览空白的问题。
- 保存配置时会把当前有效的预览 token 反解为真实本地图片路径，并自动跳过历史遗留的失效预览 token。

## 3.3.1

- 修复多组人设中较大的本地参考图刷新后只显示数量、不显示预览的问题。
- 大图预览改为通过受控 token API 回传，避免把本地路径直接交给浏览器。

## 3.3.0

- 新增多组人设档案，支持每个人设独立维护 Base Prompt 与参考图组。
- 新增 `/人设` 与 `/切换人设 [序号/ID/名称]`，可在指令侧热切换自拍人格。
- 配置页新增人设组管理，切换人设时同步切换对应参考图矩阵。
- 保持旧版单人设字段兼容，旧配置会自动迁移为第一组人设。

## 3.2.1

- 深度优化提示词副脑，强制成功优化结果使用英文输出。
- 精进内置风格矩阵，尤其强化“自拍专用极致真实”的日常手机自拍质感。
- 增加人体结构、自拍角度、皮肤质感、反光控制、畸形规避等真实感护栏。

## 3.2.0

- 统一插件版本、作者和 metadata 注册信息。
- 按 AstrBot 数据目录规范持久化配置与参考图。
- 增强配置归一化，兼容旧模型逗号写法和多参考图。
- 修复副脑链路保存位置、空 API Key 崩溃、视频参数丢失和 URL 拼接问题。
- 增加后台视频任务生命周期清理。
- 修复配置页潜在 HTML 注入、预设冒号截断和桌面端卡片样式缺失。
